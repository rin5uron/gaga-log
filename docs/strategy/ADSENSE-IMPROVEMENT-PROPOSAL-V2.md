## 2. 原因分析（自己分析）

### 2.1 最も疑わしい原因：量産型サイトに見える構造

#### ① H2見出しが全記事で同一

全35記事で以下のH2構造が完全に統一されている：

```markdown
# [曲名]

## About この曲について
## Lyrics 歌詞のポイント
## Favorite Lines 口ずさみたいフレーズ
## Music Video MVの見どころ
## Analysis 考察
```

**問題点:**
- H2はページ構造の「章タイトル」として認識される
- 全記事で同一のH2が並ぶ → **「テンプレートに文字を流し込んだだけ」に見える**
- AI生成コンテンツや量産サイトの判定基準と重なる

#### ② テンプレート構造が完全に統一

- 見出し構成・流れ・文体がほぼ同一
- 内容は違っていても、構造的に「同一フォーマットで大量生成されたページ群」と判断される可能性

#### ③ H1（曲名）も全記事で似た形式

```html
<h1>Replay</h1>
<h1>Shallow</h1>
<h1>Telephone</h1>
```

- H1が曲名だけ → 記事ごとの独自性がHTML構造から伝わらない
- SEO的にも、H1がページの主題を表していない

### 2.2 今回は原因として考えにくい点（除外）

以下は改善済み、または該当しない：

- ❌ 文字数不足：平均4,000〜7,000文字（十分）
- ❌ 歌詞引用の多さ：1〜2行のフレーズのみ（かなり抑えている）
- ❌ 著作権違反：公式YouTube埋め込み＋最小引用（問題なし）
- ❌ 広告過多：致命的な配置ではない

## 3. 改善方針

### 3.1 核心：量産型サイト構造からの脱却

**目標**: 人が一曲ずつ考えて書いているサイトと、構造から一瞬で分かるようにする。

### 3.2 改善の3本柱

| 改善項目 | 目的 | 優先度 |
|----------|------|--------|
| ① H1の改善 | 記事ごとに異なるH1にする | 🔴 最優先 |
| ② 考察テーマの明示 | 各記事の「観点」を明確にする | 🔴 最優先 |
| ③ H2の多様化 | テンプレート感を消す | ⚠️ 高 |

## 4. 改善策の詳細

### 4.1 H1の改善：曲名 + 考察テーマ

#### 現状の問題

```html
<h1>Replay</h1>
```

- 全記事でH1が曲名のみ
- HTML構造上、記事ごとの独自性が伝わらない

#### 改善後

```yaml
# frontmatter
title: "Replay"
theme: "Chromaticaアルバムの核心、トラウマと向き合う楽曲"
```

```html
<h1 class="page-title">
  <span class="song-name">Replay</span>
  <span class="theme">Chromaticアルバムの核心、トラウマと向き合う楽曲</span>
</h1>
```

#### メリット

- **HTML構造上、全記事で異なるH1** → 量産サイト判定を回避
- **SEO**: H1がページの主題を明確に表す
- **見た目**: 曲名は大きく、themeは小さく表示（デザインはシンプルなまま）
- **主観を全面に出さない**: 「私が思う」ではなく、「この記事のテーマ」として客観的

#### 他の記事の例

```yaml
# shallow.md
title: "Shallow"
theme: "映画『アリー/ スター誕生』を象徴する、深さと表層の対比"

# telephone.md
title: "Telephone"
theme: "Beyoncéとの共演で実現した、自由への逃避"

# bad-romance.md
title: "Bad Romance"
theme: "破壊的な愛を、ファッションとアートで表現した楽曲"
```

---

### 4.2 考察テーマの明示

#### 方針

各記事を「1テーマ＝1観点」に絞った考察ページにする。

- Before: 曲の情報を網羅的にまとめたページ（辞書型）
- After: 「この曲を〇〇という観点で語る」ページ（考察型）

#### メリット

- **差別化**: 和訳サイト・歌詞解説サイトとの決定的な違い
- **人間性**: 「人が考えて書いている」ことが構造から伝わる
- **深掘り**: テーマを絞ることで、より深い考察が可能

---

### 4.3 H2とラベルの構造変更（方針確定）

#### 現状の問題

全記事で以下のH2が統一：

```markdown
## About この曲について     ← H2（全記事で同じ→量産感）
### ズートピア2の劇中歌——  ← H3
### エド・シーランが...      ← H3
## Lyrics 歌詞のポイント    ← H2（全記事で同じ）
### Try Everythingとの違い  ← H3
```

#### 改善後（確定方針）

- **ラベル**（「この曲について」「歌詞のポイント」など）→ **H2にしない**。スタイルとして表示
- **現在のH3** → **H2に昇格**
- **記事の流れは今のまま**（About → Lyrics → Favorite Lines → MV → Analysis → References）
- **ラベルは残す**（読者にとってわかりやすい）

```markdown
[この曲について]           ← ラベル（スタイル、見出しではない）
## ズートピア2の劇中歌——   ← H2（記事ごとに違う）
## エド・シーランが...      ← H2
[歌詞のポイント]           ← ラベル（スタイル）
## Try Everythingとの違い    ← H2
```

#### メリット

- **H2が記事ごとに異なる** → 量産サイト判定を回避
- **ラベルは残る** → 読者にとってわかりやすい
- **流れは同じ** → 実装の負担が少ない

## 9. まとめ

### 問題の本質

「有用性の低いコンテンツ」と判断された理由は、**中身ではなく構造の見え方**。

- 歌詞引用は最小限
- 文字数も十分
- 独自の考察もある

**→ それでも落ちた**

### 改善の核心

**HTML構造上の独自性を明確にする**

1. **H1の改善**: 曲名 + 考察テーマで、全記事で異なるH1にする
2. **テーマ明示**: 各記事を「1テーマに絞った考察ページ」にする
3. **H2の多様化**: 記事ごとに異なるH2で、量産感を消す

### 期待される効果

- 量産サイト判定の回避
- 和訳サイト・歌詞解説サイトとの差別化
- SEOの改善（H1がページの主題を明確に表す）
- ユーザー体験の向上（各記事のテーマが明確）

# 開発用語集

## ブラウザ関連

| 用語 | 説明 | 簡単な例え |
|------|------|------------|
| **ハードリロード** | ブラウザのキャッシュ（保存されたデータ）を無視して、最新の状態を読み込むこと | 古い写真を捨てて、新しい写真を撮り直す感じ |
| **キャッシュ** | ブラウザが以前のデータを保存して、表示を速くする仕組み | よく使うものを手元に置いておく引き出し |
| **404エラー** | ページやファイルが見つからないというエラー | 本棚に本を探しに行ったけど、その本がなかった |

## サーバー・開発環境関連

| 用語 | 説明 | 簡単な例え |
|------|------|------------|
| **ローカルホスト（localhost）** | 自分のパソコン上で動いているサーバーのアドレス | 自分の家の住所みたいなもの |
| **ポート番号** | サーバーがどの「窓口」で待っているかを示す番号 | 家のドア番号みたいなもの（3000、3001など） |
| **開発サーバー** | コードを書きながら、すぐに結果を見られるようにする仕組み | 料理を作りながら、味見ができるキッチン |
| **ビルド** | コードを実際に動かせる形に変換すること | 設計図から実際の家を建てること |

## Next.js関連

| 用語 | 説明 | 簡単な例え |
|------|------|------------|
| **Next.js** | Reactという技術を使って、ウェブサイトを作りやすくする道具 | 家を建てるための便利な道具セット |
| **静的サイト生成（SSG）** | あらかじめページを作っておいて、それを表示する方法 | 本を印刷してから売る感じ |
| **動的ルート** | URLの一部が変わるページ（例：`/posts/hey-girl`） | 同じテンプレートで、違う内容を表示するページ |

## エラー関連

| 用語 | 説明 | 簡単な例え |
|------|------|------------|
| **favicon.ico** | ブラウザのタブに表示される小さなアイコン | 本の表紙の小さなイラスト |
| **500エラー（Internal Server Error）** | サーバー側でエラーが発生したというエラー | レストランで料理を作ろうとしたら、キッチンで問題が起きた |
| **型エラー** | コードの書き方が間違っているというエラー | 日本語の文章で、文法が間違っている感じ |
| **コンパイルエラー** | コードを変換するときに見つかったエラー | 翻訳するときに、意味がわからない言葉があった |

## よくある操作

| 操作 | やり方 | いつ使う？ |
|------|--------|------------|
| **ハードリロード** | Mac: `Cmd + Shift + R`<br>Windows: `Ctrl + Shift + R` | 変更が反映されないとき |
| **開発者ツールを開く** | `F12`キーを押す | エラーを確認したいとき |
| **開発サーバーを起動** | `npm run dev` | コードを書き始めるとき |
| **ビルド** | `npm run build` | 本番環境にデプロイする前 |

## Next.jsを使う理由

### 良い点
- **自動でページを作ってくれる**：記事を追加するだけで、自動的にページが作られる
- **速い**：あらかじめページを作っておくので、表示が速い
- **SEOに強い**：検索エンジンが見つけやすい
- **開発が楽**：コードを書くと、すぐに結果が見られる

### 大変な点
- **エラーが出やすい**：設定が複雑で、エラーが出ることがある
- **学習コスト**：覚えることが多い

### このプロジェクトでNext.jsを使う理由
- 記事（Markdownファイル）を追加するだけで、自動的にページが作られる
- ブログのようなサイトを作るのに向いている
- 無料でVercelにデプロイできる

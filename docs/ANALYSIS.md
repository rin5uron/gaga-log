# プロジェクト将来性分析

**作成日**: 2024年12月27日  
**最終更新**: 2024年12月27日

---

## 🔍 水平思考バイアスとは？

「似たような成功例があるから、これも成功するはず」という安易な推論のことです。

**例**: 「歌詞解説サイトは人気がある → このサイトも人気が出る」

このバイアスを避けて、**このプロジェクト固有の条件**を冷静に見ていきます。

---

## ✅ 強み（本質的に価値があるポイント）

### 1. 「売らない」という設計が逆に強い

- 多くの歌詞サイトは「SEO狙い」「PV稼ぎ」が透けて見える
- **「自分のために残す」という姿勢は、実は最も信頼される声**
- これは模倣が難しい（AIでは再現できない）

### 2. 著作権リスクを最小化している

- 1記事1フレーズ・90文字未満は**非常に堅実**
- 歌詞全文掲載サイトが訴訟リスクを抱える中、この設計は**長期運営向き**

### 3. AdSenseとの相性

- 滞在時間が長い読者 → 広告価値が高い
- PVは少なくても、**深く読む人が来るなら収益化の可能性あり**

### 4. 「感情ログ」という独自性

- 既存の歌詞サイトは「意味の解説」「和訳」ばかり
- **「なぜこの言葉に惹かれたか」は空白地帯**
- 競合が少ない = 検索で拾われやすい可能性

---

## ⚠️ 弱み・リスク（水平思考バイアスを排除して見る）

### 1. 「ニッチすぎて発見されない」可能性

- **バイアス例**: 「深い記事なら見つけてもらえる」
- **現実**: SEOで上位に出なければ、どんなに良い記事でも読まれない
- 特に洋楽フレーズは競合が強い（Genius、UtaTen、個人ブログ多数）

**対策案**:
- ロングテール検索を狙う（「〇〇 フレーズ 由来 なぜ」など）
- 最初から検索流入を期待しない設計（SNS拡散、自己アーカイブ優先）

### 2. 「共感される前提」の危うさ

- **バイアス例**: 「同じ温度の人が読んでくれるはず」
- **現実**: その「同じ温度の人」がサイトにたどり着く導線がない
- 「深い記事を書けば自然と読者が来る」は**最も危険な思い込み**

**対策案**:
- 最初の10記事はSNS（X/Threads）で自ら発信
- 「読まれなくてもOK」を本当に受け入れる覚悟を持つ

### 3. AI半自動の限界

- **バイアス例**: 「AIが整えてくれるから量産できる」
- **現実**: 「温度」や「揺れ」は人間が毎回書かないと記事にならない
- AI下書きに頼りすぎると、結局「AIっぽい記事」になりやすい

**対策案**:
- AI使用は構成整理だけに限定
- 核心部分（なぜ惹かれたか）は必ず自分で書く

### 4. 継続性の壁

- **バイアス例**: 「好きなことだから続けられる」
- **現実**: 反応がゼロの状態で1年続けるのは非常に難しい
- 「おまけ」と言っても、人間は数字を見てしまう

**対策案**:
- 記事数の目標を設定しない（「週1本」などのノルマは挫折の元）
- 「書きたい時だけ書く」を本当に許可する

---

## 📊 将来性の見立て（正直に）

| 観点 | 評価 | 理由 |
|------|------|------|
| **収益性（AdSense）** | △ 低め | ニッチすぎてPVが伸びにくい。月数千円レベルが現実的 |
| **長期運営可能性** | ◎ 高い | 著作権リスクが低く、燃え尽きにくい設計 |
| **独自性** | ◎ 高い | 「感情ログ」という切り口は空白地帯 |
| **発見されやすさ** | △ 低め | SEO競合が強い領域。導線設計が必要 |
| **自己満足度** | ◎ 高い | 本質的に「自分のための記録」なので価値がある |

---

## 💡 最終的な見解

### このプロジェクトが成功する条件

1. **「誰にも読まれなくてもOK」を本気で受け入れられるか**
   - これが嘘でなければ、絶対に続く
   - 少しでも「読まれたい」があるなら、導線設計が必須

2. **「アーカイブとして完成している」状態を目指す**
   - 1年後、記事が30本あれば、それ自体が価値
   - 検索に引っかかるようになるのは、記事が溜まってから

3. **最初の目標を「自分が読み返して嬉しいか」に置く**
   - 他者評価を入れると、必ずブレる

## 🚀 具体的な次のステップ提案

このリポジトリで実現するなら、技術的には以下が考えられます：

1. **静的サイトジェネレーター**（Hugo / Astro / Next.js）でシンプルに構築
2. **Markdownで記事管理**（AIとの連携がしやすい）
3. **Vercel / Netlify で無料ホスティング**
4. **最小限のデザイン**（記事に集中するため）

# Markdown記法ガイド（gaga-log用）

このブログで使えるマークダウン記法をまとめました。

---

## 📌 基本構造

### 見出し（Headings）

```markdown
# H1見出し（記事タイトル専用、使わない）
## H2見出し（大見出し）
### H3見出し（中見出し）
#### H4見出し（小見出し、使わない推奨）
```

**出力:**
```html
<h1>H1見出し</h1>
<h2>H2見出し</h2>
<h3>H3見出し</h3>
<h4>H4見出し</h4>
```

**このブログでのルール:**
- ❌ `#`（H1）は使わない → 記事タイトルが自動的にH1になる
- ✅ `##`（H2）を大セクション区切りに使う
- ✅ `###`（H3）をサブセクションに使う
- ❌ `####`（H4）は使わない → Note風のシンプルさが失われる
  - 代わりに**太字**を使う

## 🔤 テキスト装飾

### 太字・斜体

```markdown
**太字テキスト**
*斜体テキスト*
***太字+斜体***
```

**出力:**
```html
<strong>太字テキスト</strong>
<em>斜体テキスト</em>
<strong><em>太字+斜体</em></strong>
```

**このブログでのルール:**
- ✅ 強調したい単語・フレーズに`**太字**`を使う
- ❌ 斜体（`*`）はほぼ使わない → 日本語では読みにくい

---

## 📝 段落と改行

### 段落

```markdown
これは段落1です。

これは段落2です。
```

**ルール:**
- 段落と段落の間には**空行（1行）**を入れる
- 空行なしで改行しても、HTMLでは1つの段落になる

### 改行（強制）

```markdown
1行目<br>
2行目
```

**出力:**
```html
1行目<br>
2行目
```

**このブログでのルール:**
- ✅ 詩的な表現や短い文章の連続に`<br>`を使う
- ❌ 通常の文章では使わない → 空行で段落を分ける

## 🔗 リンク

### 内部リンク（サイト内の記事）

```markdown
[リンクテキスト](/posts/記事のslug)
```

**例:**
```markdown
[The MAYHEM Ball Tour](/posts/mayhem-ball-tour)
```

**このブログでのルール:**
- ✅ `/posts/`で始まる相対パスを使う
- ✅ `relatedPosts`に設定した記事へのリンクを本文に入れる

### 外部リンク

```markdown
[リンクテキスト](https://example.com)
```

**例:**
```markdown
[Lady Gaga公式サイト](https://www.ladygaga.com)
```

**このブログでのルール:**
- ✅ 公式サイト・公式情報へのリンクに使う
- ✅ `target="_blank"`は自動的に付与される

## 📋 リスト

### 順序なしリスト（Unordered List）

```markdown
- 項目1
- 項目2
  - ネスト項目2-1
  - ネスト項目2-2
- 項目3
```

**出力:**
```html
<ul>
  <li>項目1</li>
  <li>項目2
    <ul>
      <li>ネスト項目2-1</li>
      <li>ネスト項目2-2</li>
    </ul>
  </li>
  <li>項目3</li>
</ul>
```

**このブログでのルール:**
- ✅ チェックリスト、情報の列挙に使う
- ✅ ネストは2階層まで

### 順序付きリスト（Ordered List）

```markdown
1. 項目1
2. 項目2
3. 項目3
```

**出力:**
```html
<ol>
  <li>項目1</li>
  <li>項目2</li>
  <li>項目3</li>
</ol>
```

**このブログでのルール:**
- ✅ セットリスト、手順、ランキングに使う

## 🖼️ 画像

### 基本構文

```markdown
![代替テキスト](画像のパス)
```

**このブログでのルール:**
- ❌ マークダウン記法は使わない
- ✅ HTMLで記述する（キャプション、サイズ制御のため）

### HTMLでの画像挿入（推奨）

```html
<div style="margin: 1.5rem 0;">
  <img src="/images/フォルダ名/ファイル名.jpg" alt="代替テキスト" style="width: 100%; max-width: 600px; border-radius: 8px;">
  <p style="font-size: 0.875rem; color: #6b7280; margin-top: 0.5rem;">キャプションテキスト</p>
</div>
```

**ルール:**
- ✅ `alt`属性は必ず記述（SEO対策）
- ✅ キャプションは必須（ユーザー体験向上）
- ✅ `max-width: 600px`でサイズ制限

## ➖ 水平線

### 構文

```markdown
---
```

**出力:**
```html
<hr>
```

**このブログでのルール:**
- ⚠️ **通常は`display: none`で非表示**（`.prose hr { display: none; }`）
- ⚠️ **`.info-card`内では目次に影響するため注意**
  - 現在はCSSで非表示にし、代わりに疑似要素で線を生成

**使用例:**
```markdown
## セクション1

本文...

---

## セクション2
```

→ **目次に影響するため、大セクション間の区切りにのみ使用**

## 🎨 このブログ専用のコンポーネント

### 情報カード（.info-cards-container）

```html
<div class="info-cards-container">

<div class="info-card">

<strong>【カードタイトル】</strong>

カード内の本文...

**見出し1**

説明文...

**見出し2**

説明文...

</div>

<div class="info-card">
<!-- 2枚目のカード -->
</div>

</div>
```

**ルール:**
- ✅ 縦1列で表示される
- ✅ 2番目以降の`**太字**`の前に自動的に灰色の線が表示される
- ❌ カード内で`---`を使わない（目次に影響する）

### チェックリストボックス（.checklist-box）

```html
<div class="checklist-box">

#### タイトル

- **項目1**（説明）
- **項目2**（説明）

</div>
```

**用途:** 持ち物リスト、服装チェックリストなど

### 公式情報ボックス（.official-info-box）

```html
<div class="official-info-box">

#### タイトル

- **項目1** - 説明
- **項目2** - 説明

</div>
```

**用途:** 公式発表、注意事項など

### タイムラインボックス（.timeline-box）

```html
<div class="timeline-box">

#### 時間配分の目安

- **2時間前**: 説明
- **1時間前**: 説明

</div>
```

**用途:** 時間配分、スケジュール案内

### まとめセクション（.summary-plain）

```html
<div class="summary-plain">

**まとめ**

まとめの本文...

</div>
```

**用途:** 記事の最後のまとめ

## ⚠️ マークダウンの落とし穴

### 1. `---`は必ず`<hr>`になる

```markdown
---
```

→ **常に水平線（`<hr>`タグ）として解釈される**
→ **目次生成に影響する可能性がある**

**対処法:**
- `.info-card`内では使わない
- 大セクション間の区切りにのみ使用

### 2. 空行なしの改行は無視される

```markdown
1行目
2行目
```

→ **1つの段落として結合される**

**対処法:**
- 段落を分けたい場合は空行を入れる
- 強制改行したい場合は`<br>`を使う

### 3. インデントでコードブロックになる

```markdown
    これはコードブロック
```

→ **4スペースまたは1タブでインデントすると`<pre><code>`になる**

**対処法:**
- 意図しない場合はインデントしない
- コードブロックを明示する場合は` ```言語名 `を使う

### 4. `#`の後にスペースが必要

```markdown
##見出し  ← ❌ 見出しにならない
## 見出し ← ✅ 正しい
```

### 5. リストの前後には空行が必要

```markdown
本文...
- 項目1  ← ❌ リストにならない

本文...

- 項目1  ← ✅ 正しい
```

</div>

```

# H1は使わない

#### H4も使わない

本文の改行は
このようにしない（空行なし）

---
---
---
← 水平線を連続で使わない

**見出し**
---
← カード内で---を使わない

</div>
```

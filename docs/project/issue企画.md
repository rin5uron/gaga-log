# Issue管理企画

## 基本方針

### Issue作成のルール

1. **ページ（記事）ごとにIssueを作成**
   - 例: `[楽曲] Bad Romance - SEO改善・内容追加`
   - 例: `[ライブ] The MAYHEM Ball Tour - 情報更新`
   - 例: `[映像] Gaga: Five Foot Two - レビュー追加`

2. **Issue内で改善履歴を管理**
   - 初期作成時のチェックリスト
   - SEO改善の記録（キーワード、メタデータ）
   - 内容追加・修正の履歴
   - コメントで進捗・判断理由を残す

3. **ラベルで分類**
   - **コンテンツタイプ**: `song` / `live` / `movie` / `other`
   - **作業内容**: `seo` / `content` / `fix` / `enhancement`
   - **優先度**: `priority-high` / `priority-medium` / `priority-low`
   - **ステータス**: `draft` / `in-progress` / `review` / `done`

---

## Issue作成タイミング

### 必ず作成するケース
- 新規記事を作成する時
- 大きなSEO改善を行う時
- コンテンツの大幅な追加・修正を行う時
- ライブレビューなど未来のタスク

### 作成しなくてもいいケース
- 誤字脱字の修正
- リンク切れの修正
- 軽微なスタイル調整

## Issue運用ガイドライン

### Issueを開く時
1. テンプレートを使用
2. ラベルを必ず設定
3. 関連Issueがあればリンク

### Issue進行中
1. 進捗をコメントで報告
2. 判断理由を記録（なぜこのキーワードを選んだか、など）
3. 完了したチェックリストにチェック

### Issueを閉じる時
1. 全チェックリストが完了していることを確認
2. 最終的な成果物へのリンクを記載
3. 学びや次回への改善点をコメント

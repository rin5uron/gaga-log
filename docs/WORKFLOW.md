# ワークフロー - gaga-log

## 記事作成フロー

### PHASE 1: 準備
1. **曲を選ぶ**: 自分が深掘りしたい曲、ライブで歌いたい曲
2. **リサーチ**: 公式インタビュー、歌詞、MVの意味を調べる
3. **テンプレート確認**: `docs/templates/` の記事テンプレートを確認

### PHASE 2: 執筆
1. **コマンド実行**: `/new-article` で新規記事を作成
2. **Front Matter 設定**:
   - `title`: 記事タイトル（SEOキーワード含む）
   - `description`: 検索結果に表示される説明文（120文字以内）
   - `highlights`: この記事でわかること（2〜3点、具体的に）
   - `date`: 公開日
   - `artist`: アーティスト名
   - `song`: 曲名
   - `category`: カテゴリ（song, live, artist）
3. **本文執筆**:
   - **基本スタンス**: 解釈・考察（感想文ではない）
   - **構成**: 導入 → 曲の背景 → 歌詞のポイント → まとめ
   - **参考情報**: 公式リンク、ストリーミング、関連記事を記載

### PHASE 3: SEOチェック
1. **コマンド実行**: `/seo-check` でSEO最適化チェック
2. **確認項目**:
   - title/descriptionにキーワード含まれているか
   - 見出し構造（h2/h3）が正しいか
   - 内部リンク（関連記事）が適切か

### PHASE 4: 公開
1. **ビルド**: `npm run build` でエラーチェック
2. **デプロイ**: Vercelに自動デプロイ
3. **Search Console**: 新規ページをインデックス登録

## 記事構成とデザイン要素

### 🎯 この記事でわかること（highlights）

**目的**: 読者が「読む価値」を一目で判断できる

**書き方ルール**:
- **2〜3点まで**（焦点を絞る）
- **具体的＋簡潔**: 固有名詞を入れ、一文で意味が通じる
- **SEOキーワード含む**: 検索されそうな言葉を入れる

**良い例**:
```yaml
highlights:
  - ガガとビヨンセの共演背景
  - Paparazziから続く"超大作MV"の考察
```

**悪い例**:
```yaml
highlights:
  - 「Paparazzi」が描く名声の光と影——愛と執着の境界線（長すぎる）
  - この曲の魅力について（曖昧）
```

### 📖 目次（TableOfContents）

**特徴**:
- アコーディオン式で閉じられる（モバイルでコンテンツを圧迫しない）
- h2はセクション見出し、h3はクリック可能
- スクロール連動でアクティブな見出しをハイライト
- 参考情報セクション以降は非表示

### 🔗 参考情報セクション

**目的**: 信頼性向上（E-E-A-T対策、AdSense要件）

**必須項目**:
- 公式リンク（公式サイト、Wikipedia、Genius Lyrics）
- ストリーミングリンク（Spotify、Apple Music、YouTube Music）
- 関連記事（同じアーティストの他の曲、関連アルバム）

## コマンド詳細

### `/new-article`
新規記事を作成（テンプレート準拠・Front Matter設定まで）

**実行内容**:
1. テンプレートファイルを読み込み
2. Front Matterを設定（title, description, highlights, date, artist, song, category）
3. 基本構造を生成（導入、曲の背景、歌詞のポイント、参考情報）
4. ファイル保存（`content/posts/[slug].md`）

### `/seo-check`
SEO最適化チェック（Search Consoleデータを元に改善提案）

**実行内容**:
1. Search Consoleデータを確認（CTR、表示回数、順位）
2. title/descriptionを分析
3. 改善提案を提示
4. ユーザー承認後に修正

### `/seo-update`
追加SEO改善（効果測定後の再改善）

**実行内容**:
1. 前回のSEO改善効果を確認
2. さらなる改善点を提案
3. 内部リンク強化、見出し最適化など

## 毎日の作業ルーチン（30分）

### Phase 1: データ分析（5分）
1. Search Console/Analytics確認
2. CTR 3%未満、表示回数100以上だがクリック10未満のページを特定
3. Issue提案（label: seo）

### Phase 2: 記事改善（10分）
1. Issue（label: seo）を確認
2. 10分以内で完了できるタスクを選定
3. 実装（title/description最適化、内部リンク追加など）
4. Issueを更新

### Phase 3: 新規記事作成（10分）
1. 新規記事の曲を選ぶ
2. `/new-article` で執筆
3. 完成度60%でOK（まず公開して、後から磨き上げる）

### Phase 4: Next.js課題（5分）
1. `docs/learning/nextjs-progress.md` を確認
2. 5分以内で完了できる学習タスクを実装
3. 学習進捗を更新

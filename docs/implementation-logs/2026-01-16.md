## 実装内容

### 1. ZOO記事の見出し改善

**改善方針:**
- 見出しだけで内容が分かる構造に変更
- 海外翻訳されても意味が破綻しない表現に統一
- 疑問形・感情的表現を避け、中立的で具体的な見出しに

**変更例:**
- `なぜエド・シーランがディズニー映画を？` → `なぜエド・シーランがディズニー映画を作曲したのか`
- `MVに歌詞付き！みんなで歌える工夫` → `歌詞付きMV：みんなで歌える工夫`
- `楽しさを、そのまま受け取る` → `この曲の楽しみ方`

**コミット:** `5d5b6c5`

---

### 2. タイトルの簡潔化

**対象記事:**
- ZOO: `Zoo（ズートピア2主題歌）歌詞の意味・和訳｜...` → `Zoo`
- LoveDrug: `Lovedrug（レディー・ガガ）歌詞の意味・和訳｜...` → `LoveDrug`
- Aura: `Aura（レディー・ガガ）歌詞の意味・和訳｜...` → `Aura`

**方針:**
- シンプルに曲名のみ
- 詳細情報はdescriptionで補足
- 他の記事との統一性を保つ

**コミット:** `b929112`, `da29923`

---

### 3. 見出しに視覚的区切り（帯）を実装

**デザイン仕様:**
- **h2（主要セクション）**: 青い縦線 4px (`border-left: 4px solid #3b82f6`)
- **h3（サブセクション）**: グレーの縦線 3px (`border-left: 3px solid #94a3b8`)

**効果:**
- 階層が一目で分かる
- 流し読みしやすい
- h2とh3の視覚的差別化

**コミット:** `b929112`

---

### 4. 余白・引用デザイン改善

**見出しの余白を増やす:**
- h2: `margin-top 3rem→5rem`, `margin-bottom 1rem→2rem`
- h3: `margin-top 2.5rem→4rem`, `margin-bottom 0.4rem→1.5rem`
- h2直後のh3: `margin-top 1rem→1.5rem`

**引用（blockquote）デザイン改善:**
- 左の縦線を削除（見出しとの差別化）
- `display: inline-block` + `max-width: fit-content` で中身にフィット
- 四隅すべてに角丸適用

**効果:**
- セクション間が明確に区切られ、文字詰まりを解消
- 引用が見出しと混同されない
- 引用の幅が内容に応じて調整される

**コミット:** `da29923`

### 5. READMEに記事執筆の基本ルールを追加

**追加セクション:** `## 記事執筆の基本ルール`

**やらないこと:**
- ❌ 和訳・意訳はしない
- ❌ 逐語訳で解説しない
- ❌ 正解を押し付けない

**やること:**
- ✅ 原文（英語など）を引用する
- ✅ 音と雰囲気を記録する
- ✅ 自分の感じたことを書く
- ✅ 流し読みできる構成

**タイトル・見出しのルール:**
- タイトルはシンプルに曲名のみ
- 見出しだけで内容が分かる
- 疑問形は避ける
- 翻訳されても意味が通る

**理由:**
- サイトのコンセプト「音と雰囲気を記録する場所」を明文化
- AI・人間が記事を書く際の統一基準を明確化

**コミット:** `33536e7`

**改善内容:**

- 「自分の感じたこと」に書き換え
- 見出しを具体的に改善（h3追加で階層化）

**Before:**
```markdown
「腰は嘘をつかない。そして、正しいと感じ始めている」
```

**After:**
```markdown
"Hips don't lie"——このフレーズが曲中で何度も繰り返される。
シンプルで、覚えやすく、リズムに乗りやすい。
```

## 技術的な注意点

### APIエラーについて
- 著作権のある歌詞の大量引用がAPIのコンテンツフィルタリングポリシーでブロックされる
- サイト自体は問題ないが、編集中に大量の歌詞を含むテキストを扱うとエラーになる
- 対策: 歌詞引用を最小限にし、原文のみ引用、和訳は書かない

### Tailwind CSS の `@apply`
- `@apply` はTailwindのユーティリティクラスをCSSファイル内で使える機能
- 例: `@apply text-2xl font-bold;` → `font-size: 1.5rem; font-weight: 700;`

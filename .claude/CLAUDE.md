## Claude自身の行動ルール

### 絶対に守ること

1. **ユーザーの許可なくファイルを編集・コミット・issueを作成しない**
   - 変更内容を先に提示し、「やっていいですか？」と確認してから実行する
   - 「一緒にやろう」は「内容を相談しながら進める」という意味。勝手に進めてはいけない

2. **改善案は必ず先に出す**
   - リライト作業は「案を出す→確認→実行」の順番
   - 確認なしに本文を書き換えない

3. **コミット前に確認する**
   - git commit・git push は実行前に一言確認する

4. **issue作成前に内容を見せる**
   - issueの本文案をチャットに出してOKをもらってから作成する

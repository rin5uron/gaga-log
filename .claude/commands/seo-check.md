# SEO改善プロセス開始

このコマンドは、記事のSEO改善を体系的に進めるためのガイド付きプロセスです。

## プロセス

1. **SEOチェックリストを参照** - `docs/WORKFLOW.md` の「SEOチェックリスト」セクションを参照
2. **ユーザーに記事を選択してもらう** - どの記事を改善するか質問
3. **現状確認** - 記事の現在の状態を確認
4. **段階的に質問** - チェックリストに沿って質問
5. **改善案を提示** - タイトル、ディスクリプション、見出し、本文の改善案を提示
6. **承認後に実装** - ユーザーの承認を得て実装
7. **記録** - GitHub Issue に改善履歴を記録

---

## 実行手順

### ステップ1: 記事選択

まず、改善したい記事を選択してください。

**質問**: どの記事を改善しますか？

選択肢:
- 記事のスラッグ名（例: bad-romance）
- または「一覧を見せて」で全記事リストを表示

---

### ステップ2: 現状確認

選択された記事の以下を確認します:

1. **タイトル** - frontmatterの`title`
2. **文字数** - 本文の文字数をカウント
3. **見出し構造** - H2, H3の数と内容
4. **内部リンク** - `relatedPosts`の設定
5. **メディア** - YouTube埋め込み、ストリーミングリンクの有無

**ユーザーへ報告**:
```
【現状確認】
- タイトル: [タイトル]
- 文字数: [文字数]文字
- 見出し数: H2が[数]個、H3が[数]個
- 内部リンク: [数]個
```

---

### ステップ3: キーワード確認

**質問1**: この記事で狙いたいメインキーワードは何ですか？

例:
- Bad Romance MV 意味
- Bad Romance 棺桶 シーン
- Lady Gaga Bad Romance 解説

**質問2**: サブキーワードは何ですか？（2〜3個）

例:
- ファッション
- 色
- 象徴

**質問3**: 競合サイトを確認しましたか？上位10サイトに欠けている情報は何ですか？

---

### ステップ4: 改善案提示

ユーザーからの情報を元に、以下の改善案を提示します:

#### タイトル改善案（3案）
```
【現在】[現在のタイトル]

【改善案1】[疑問形パターン]
【改善案2】[数字パターン]
【改善案3】[具体性パターン]
```

#### ディスクリプション改善案
```
【現在】[現在のディスクリプション]

【改善案】[120〜160文字の改善案]
```

#### 見出し追加提案
```
【追加すべき見出し】
## [見出し1]
### [小見出し1-1]
### [小見出し1-2]
## [見出し2]
```

#### 本文加筆提案
```
【追加すべき情報】
1. [情報1]
2. [情報2]
3. [情報3]

【追加場所】
- [見出し名]のセクション
```

#### 内部リンク提案
```
【追加すべきリンク】
- [関連記事1]
- [関連記事2]
```

---

### ステップ5: 承認確認

**質問**: どの改善を実装しますか？

選択肢:
- タイトル案1を採用
- タイトル案2を採用
- タイトル案3を採用
- カスタムタイトルを入力
- ディスクリプションを採用
- 見出しを追加
- 本文を加筆
- 内部リンクを追加
- すべて実装

---

### ステップ6: 実装

承認された改善を実装します:

1. **タイトル修正** - frontmatterの`title`を更新
2. **見出し追加** - 本文に見出しを追加
3. **本文加筆** - 必要な情報を追加
4. **内部リンク追加** - `relatedPosts`を更新

---

### ステップ7: 記録

GitHub Issue に以下をコメントとして記録します:

```markdown
## SEO改善完了: [記事タイトル]

**完了日**: 2026-XX-XX

**改善前**
- タイトル: [旧タイトル]
- 文字数: [文字数]
- 見出し数: [数]
- 内部リンク数: [数]

**狙うキーワード**
- メイン: [メインKW]
- サブ1: [サブKW1]
- サブ2: [サブKW2]

**実施した改善**
- [x] タイトル修正: [新タイトル]
- [x] ディスクリプション修正
- [x] 見出し追加: [数]個
- [x] 本文加筆: [文字数]文字追加
- [x] 内部リンク追加: [数]個

**次のアクション**
- 1週間後にSearch Consoleで効果測定
- 必要に応じて `/seo-update` で追加修正
```

【参考リンク】

---

## 重要な注意事項

1. **必ずチェックリストを参照** - `docs/WORKFLOW.md` の「SEOチェックリスト」に沿って進める
2. **段階的に質問** - 一度に全部聞かず、1つずつ確認
3. **具体的な提案** - 抽象的ではなく、具体的な改善案を提示
4. **記録を忘れない** - 必ず GitHub Issue にコメントとして記録
5. **ユーザー主導** - ユーザーの承認なしに実装しない

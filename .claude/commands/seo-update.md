# SEO追加修正プロセス

このコマンドは、すでにSEO改善を実施した記事の追加修正・微調整を行うためのプロセスです。

## プロセス

1. **改善記録を確認** - `docs/SEO_IMPROVEMENT_LOG.md`から過去の改善履歴を確認
2. **ユーザーに記事を選択してもらう** - どの記事を修正するか質問
3. **効果測定結果を確認** - Search Consoleのデータを聞く
4. **追加修正内容を確認** - 何を修正したいか質問
5. **改善案を提示** - 具体的な修正案を提示
6. **承認後に実装** - ユーザーの承認を得て実装
7. **記録更新** - `docs/SEO_IMPROVEMENT_LOG.md`に追加修正履歴を記録

---

## 実行手順

### ステップ1: 記事選択

**質問**: どの記事を追加修正しますか？

選択肢:
- 記事のスラッグ名（例: bad-romance）
- または「改善履歴を見せて」で過去の改善記録を表示

---

### ステップ2: 過去の改善履歴確認

`docs/SEO_IMPROVEMENT_LOG.md`から対象記事の改善履歴を読み込みます。

**ユーザーへ報告**:
```
【過去の改善履歴】
- 改善実施日: [日付]
- メインキーワード: [キーワード]
- 実施内容:
  - タイトル修正: [新タイトル]
  - ディスクリプション修正: 完了
  - 見出し追加: [数]個
  - 本文加筆: [文字数]文字
```

---

### ステップ3: 効果測定結果確認

**質問1**: Google Search Consoleで効果測定しましたか？

**質問2**: 現在の掲載順位は何位ですか？（メインキーワード）

**質問3**: 以下の指標を教えてください:
- クリック数:
- 表示回数:
- CTR:
- 平均掲載順位:

---

### ステップ4: 追加修正内容確認

**質問**: 何を修正したいですか？

選択肢:
- タイトルを再修正
- ディスクリプションを再修正
- 見出しを追加・修正
- 本文を加筆
- 内部リンクを追加
- キーワードを変更
- その他（自由記述）

**質問**: なぜその修正が必要だと思いますか？

例:
- 掲載順位が上がらない
- CTRが低い
- 競合に負けている
- ユーザーの滞在時間が短い

---

### ステップ5: 改善案提示

効果測定結果と修正希望を元に、具体的な改善案を提示します:

#### タイトル再修正案（CTRが低い場合）
```
【現在】[現在のタイトル]
【課題】CTRが[数値]%と低い → クリック誘因が弱い

【改善案1】[より具体的なタイトル]
【改善案2】[疑問形を強化したタイトル]
【改善案3】[数字を入れたタイトル]
```

#### 見出し追加案（滞在時間が短い場合）
```
【現在の見出し】
[現在の見出しリスト]

【課題】滞在時間が短い → 情報不足の可能性

【追加すべき見出し】
## [新しい見出し1]
### [詳細な小見出し1-1]
### [詳細な小見出し1-2]
## [新しい見出し2]
```

#### 本文加筆案（掲載順位が上がらない場合）
```
【課題】掲載順位が[数値]位 → 競合に情報量で負けている可能性

【競合分析】
- 1位サイト: [情報1, 情報2, 情報3]を含む
- 2位サイト: [情報4, 情報5]を含む

【追加すべき情報】
1. [情報1] - [見出し名]のセクションに追加
2. [情報2] - [見出し名]のセクションに追加
3. [情報3] - 新規セクションとして追加

【追加文字数】約[文字数]文字
```

#### キーワード変更案（検索意図のズレがある場合）
```
【現在のメインKW】[現在のキーワード]
【課題】検索意図がズレている可能性

【新しいメインKW候補】
1. [キーワード1] - 検索ボリューム[数値]、競合度[低/中/高]
2. [キーワード2] - 検索ボリューム[数値]、競合度[低/中/高]
3. [キーワード3] - 検索ボリューム[数値]、競合度[低/中/高]

【推奨】[キーワード名] - 理由: [理由]
```

---

### ステップ6: 承認確認

**質問**: どの修正を実装しますか？

選択肢:
- タイトル案1を採用
- タイトル案2を採用
- タイトル案3を採用
- 見出しを追加
- 本文を加筆
- キーワードを変更
- すべて実装
- カスタム修正（詳細を入力）

---

### ステップ7: 実装

承認された修正を実装します:

1. **タイトル再修正** - frontmatterの`title`を更新
2. **見出し追加・修正** - 本文の見出し構造を更新
3. **本文加筆** - 必要な情報を追加
4. **キーワード変更** - 新しいキーワードに合わせて最適化
5. **内部リンク追加** - `relatedPosts`を更新

---

## 重要な注意事項

1. **改善記録を必ず確認** - 過去の改善内容を把握してから修正
2. **効果測定結果を重視** - データに基づいた修正を行う
3. **段階的に修正** - 一度に大幅に変更せず、1つずつ検証
4. **記録を更新** - 必ず`docs/SEO_IMPROVEMENT_LOG.md`に追記
5. **継続的な改善** - 1位を取るまで何度でも修正

---

## 効果測定のタイミング

- 初回改善後: 1週間後
- 追加修正後: 1週間後
- 定期チェック: 毎週日曜日

---

## 改善を中止すべき判断基準

以下の場合は、キーワード自体の見直しが必要:

- 3回以上修正しても順位が30位以下
- 検索ボリュームが極端に少ない（月10未満）
- 競合が強すぎる（1位〜10位すべて大手サイト）

→ `/seo-check` で新しいキーワード戦略を立て直す

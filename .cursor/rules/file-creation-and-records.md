# ファイル作成・記録のルール

## 記録・issue 関連

- **記録は GitHub の issue で行う。新規の .md ファイルは作らない。**
- 記録を残す依頼があったら、まず `gh issue list` で該当 issue の有無を確認する。
- 該当 issue がある場合：
  - **本文（body）**：実施内容・完了条件・エラー記録など、実装に合わせて編集する。
  - **コメント**：追加でよい場合は `gh issue comment` で追記する。
- 該当 issue がない場合のみ、issue を新規作成するかユーザーに確認する。

## ファイル作成・編集の事前確認

- 任意のファイル作成・編集前に、**既存ファイルの有無を必ず確認**する。
- 既存ファイルがある場合：内容を確認し、追加・修正箇所を明示して承認を得る。
- **新規ファイルを作成する場合も、作業前にユーザーに確認を取る。**
- 記録用の新規 .md ファイルは作らず、GitHub issue に記録する。

## 記録のフォーマット

- ユーザーが指定しない限り、既存のデータ・フォーマットを変えない。
- 既存の issue 本文がある場合は、実施内容・完了条件を「実際にやったこと」に合わせて書き換え、きちんと記録する。

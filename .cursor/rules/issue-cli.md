# Issue は必ず CLI で見る・編集する

## 絶対に守ること

- **ユーザーが issue 番号（「issue 22」「#22」「issue22」など）を言ったら、回答や編集をする前に必ず `gh issue view <番号>` を実行する。**
- リポジトリ内の .md やドキュメントから「おそらくこういう内容だろう」と推測して作業しない。**正とするのは CLI で取得した issue の本文だけ。**

## 作業の流れ

1. **見る**: `gh issue view 22` のように番号を指定して本文を取得する。
2. **編集する**: 取得した本文の「作業内容」「チェックリスト」「対象記事」に沿って編集・回答する。
3. issue の「完了」「記録」を更新する場合は、`gh issue edit` や `gh issue comment` で行う（記録用の新規 .md は作らない）。

## まとめ

- issue を言われた → まず `gh issue view <番号>` で内容を取る → その内容に基づいて作業する。これ以外の順序は禁止。

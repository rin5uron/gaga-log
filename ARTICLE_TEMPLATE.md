# 記事作成マニュアル（完全版）

## 記事作成の流れ

1. **記事の目的と読者を明確にする**
2. **frontmatter設定**（必須項目を漏れなく）
3. **「この記事でわかること」を設定**（2〜3点）
4. **YouTube埋め込み**
5. **ストリーミングリンク**
6. **記事本文**（新構成で執筆）
7. **参考情報・根拠**（記事末尾）

---

## 0. 記事作成前に考えること

### 記事の目的
- **この曲で何を伝えたいか**を明確にする
- 例：「この曲の本質は〇〇だ」「この曲が今も聞かれ続ける理由は〇〇だ」

### 想定読者
- **初めて聴いた人**：曲の背景、基本的な意味、なぜ有名なのか
- **ファン**：深い解釈、他曲とのつながり、文化的背景

### 記事のスタンス
- ❌ **感想文**：「私はこう感じました」「個人的には...」
- ✅ **解釈・考察**：「この曲は〇〇を表現している」「歌詞の〇〇は〇〇を意味する」

## 1. Frontmatter（必須）

```yaml
---
title: "曲名"
song: "曲名"  # ← 重要！トップページ表示に必須
artist: "アーティスト名"
album: "アルバム名"
year: "リリース年"
date: "YYYY-MM-DD"  # ← 記事作成日（初回公開日）
updatedDate: "YYYY-MM-DD"  # ← 更新日（構成・考察を見直した日、初回はdateと同じ）
order: 番号
type: "song"
highlights:
  - ハイライト1（SEOキーワード含む、具体的な内容）
  - ハイライト2（読者が知りたい核心）
  - ハイライト3（あれば3つ目）
---
```

### ⚠️ よくある抜け漏れ

- ❌ `song` フィールドがない → トップページに表示されない
- ❌ `date` が未来の日付 → 表示順序がおかしくなる
- ❌ `highlights` がない → ハイライトボックスが表示されない
- ❌ `updatedDate` がない → 更新日が表示されない

---

## 2. 「この記事でわかること」（highlights）

**目的**: 読む価値を最初に明示する

**書き方のコツ**:
- **2〜3点まで**（多すぎると焦点がぼける）
- **SEOキーワードを含める**
- **読者の疑問に直結**
- **具体的＋問いかけ**

例：
```yaml
highlights:
  - 「Paparazzi」が描く名声の光と影——愛と執着の境界線
  - MVの衝撃的な殺人シーンに隠されたメッセージ
```

## 4. 記事本文（統一構成）

### 構成の流れ

1. **曲の概要**（背景・位置づけ）
2. **歌詞の注目ポイント**（あなた視点）
3. **あなたの考察・解釈**
4. **文化的背景・他曲とのつながり**（あれば）

### H2: 固定（全記事共通）

```markdown
## About <span class="section-subtitle">この曲について</span>
## Lyrics <span class="section-subtitle">歌詞のポイント</span>
## Favorite Lines <span class="section-subtitle">口ずさみたいフレーズ</span>
## Music Video <span class="section-subtitle">MV見どころ</span>
## Analysis <span class="section-subtitle">考察</span>
## References <span class="section-subtitle">参考情報</span>
```

**セクションの違い：**
- **Lyrics（歌詞のポイント）**: 歌詞の意味、テーマ、構造などの「解説・考察」
- **Favorite Lines（口ずさみたいフレーズ）**: 具体的なフレーズの「引用」と、そのフレーズを覚えると曲が楽しくなる「実用的なポイント」

### H3: 基本2つずつ（具体的な見出し）

```markdown
## About <span class="section-subtitle">この曲について</span>

### [具体的な状況・疑問]（例：映画のどこで歌われている？）

[事実＋あなたの視点]
- リリース情報
- 映画・アルバムとの関係
- チャート成績

### [アーティスト・背景の切り口]（例：ガガとクーパー、二人の奇跡の共演）

[事実＋意味づけ]
- アーティストの挑戦
- 制作背景

※歌詞の直接引用は最小限に

### [繰り返されるフレーズ＋歌詞の構造]（例："In the shallow" と "I'm falling"——繰り返される2つのフレーズ）

[2つのフレーズの意味＋対比・構造＋あなたの印象]
```

## 5. 参考情報・根拠（記事末尾）

**ルール**:
- **ピンポイントの出典**：本文中に `[テキスト](URL)` でリンク
- **一覧**：References セクションにまとめる（公式MV、インタビュー、その他）
- **スタイル**：参考文献セクションはフォント・サイズ・明るさを抑え、存在感を消す（CSS適用）

```markdown
---

## References <span class="section-subtitle">参考情報</span>

### 公式MV

- [曲名 - Official Music Video](URL)

### インタビュー・発言

- [インタビュー記事タイトル](URL)
- [アーティスト本人の発言](URL)

### その他

- [参考資料](URL)

---
```

## ルール

### 必須要素
- [x] frontmatter（`song`, `artist`, `date`, `updatedDate` 必須）
- [x] highlights（2〜3点）
- [x] YouTube埋め込み
- [x] ストリーミングリンク
- [x] H2の英語＋日本語サブタイトル
- [x] 区切り線 `---`
- [x] 参考情報セクション（あれば）

### H3の原則
- **基本2つずつ**（必要に応じて3つまで）
- **具体的・状況が浮かぶ見出し**
- **「私の感想」は余白で区切る**（H3だが視覚的に分離）

### 「この記事でわかること」（highlights）のルール
- **2点まで**に絞る
- **抽象語を使わない**。曲名・固有名詞を入れる
- **一文で意味が通じる**、簡潔で一目で分かること
- 例：❌「メディア性の光と影」→ ✅「『Paparazzi』というタイトルに隠された意図」

### 記事のスタンス
- ❌ **感想文**：「私はこう感じました」「個人的には...」
- ✅ **解釈・考察**：「この曲は〇〇を表現している」「歌詞の〇〇は〇〇を意味する」

### 引用ルール
- **歌詞引用**：帯なし。シンプルなインデントのみ（`<blockquote>` は左余白・イタリックのみ、背景・左边框は使わない）
- 引用を使う条件：歌詞の「重要なフレーズ」を紹介するとき。長文引用は避ける

### リンクルール
- **外部リンク（IMDb・公式など）**：原則、**本文中の該当文言に直接リンク**。（IMDb）と括弧で出すより、「ガガと共にコンセプトを手掛けた」など該当句にリンク
- **参考文献欄**：事実確認・出典の一覧のみ。本文でリンクしたものもここにまとめてOK
- **内部リンク（自サイト）**：**文脈上「次に読むと自然なもの」だけ**。アーティスト・relatedPosts は出しすぎない

### リンク・装飾（サイト内）
- **アーティスト・アルバム名**：記事内で**初出のみ**リンク。2回目以降はリンクなし
- **流し読み**：見出し＋**太字**でメリハリ。**太字だけ追っても内容が理解できる**ようにする

### AdSense対策チェックリスト
- [ ] 文字数2,000文字以上（推奨8,000文字以上）
- [ ] 歌詞の直接引用は最小限
- [ ] 出典明記
- [ ] 独自の視点・解釈を含む

## トラブルシューティング

### トップページに表示されない
- `song` フィールドがあるか確認
- `date` が未来の日付になっていないか確認
- 開発サーバーを再起動（`rm -rf .next && npm run dev`）

### ハイライトが表示されない
- YAMLの文法エラーがないか確認
- ダブルクォートの中にダブルクォートがないか確認

### アーティスト名が表示されない
- `artist` フィールドがあるか確認
- コラボ曲の場合、`&` または `/` で区切る

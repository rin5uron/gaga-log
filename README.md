# How Sound Feels - 音楽と言葉の記録サイト

**音を慈しむ。声を愛する。**

## このサイトのコンセプト

音を慈しむ。声を愛する。

その延長線上に、このサイトはあります。

歌を知ること、アーティストを知ること、その人と良い距離でいること——
それが、心地よく愛するための方法だと思っています。

このサイトは、歌詞の意味を解説するだけでなく、
**その言葉、音、雰囲気、そうしたものを記録する場所**です。

愛しさを追求するために、残す。

<br />

## プロジェクトの位置づけ

- 歌詞の意味を説明する場所ではない
- 正解を提示する場所でもない
- **その言葉、音、雰囲気、そうしたものを記録する場所**

他人に向けて"売る"のではなく、自分のために"残す"。結果として、同じ温度の人が読む。

<br />

## 記事執筆の基本ルール

### やらないこと
- ❌ **和訳・意訳はしない** - このサイトは「歌詞の意味を説明する場所ではない」
- ❌ **逐語訳で解説しない** - 言葉の正確な翻訳ではなく、音と雰囲気を記録する
- ❌ **正解を押し付けない** - 「この歌詞の意味はこうです」という断定的な説明はしない

### やること
- ✅ **原文（英語など）を引用する** - 日本語だけでなく、原文の響きを大切にする
- ✅ **音と雰囲気を記録する** - その曲を聴いたときの感覚、空気感を言葉にする
- ✅ **自分の感じたことを書く** - 「私にはこう聞こえた」「こう感じた」という主観を大切に
- ✅ **流し読みできる構成** - 見出しだけで価値が伝わる、途中から読んでも理解できる

### タイトルのルール
- **シンプルに曲名のみ** - 例: `Zoo`, `Die With a Smile`, `Bad Romance`
- 詳細情報はdescriptionで補足する
- 他の記事との統一性を保つ

### 見出しのルール
- **見出しだけで内容が分かる** - 抽象的・感情的な表現を避け、具体的に
- **疑問形は避ける** - 「〜とは？」ではなく「〜の理由」「〜の意味」
- **翻訳されても意味が通る** - 海外からのアクセスでも理解できる構造
- **目次が要約になる** - 見出しを読むだけで記事の価値が伝わる

## 🎯 このサイトの本質

### 💡 Will（やりたいこと・使命）
**音楽を深掘りすることで、豊かな人生を送る**
- 日々音楽に触れる習慣を作る
- 自分の好きな音楽をもっと深く愛せるようになる

### 🎯 Need（社会のニーズ）
**音楽を深掘りして、ライブや生活を豊かにしたい人がいる**
- 表面的な和訳サイトしかない
- 「読んで、もっと好きになる」体験が不足

### ⚙️ Can（あなたができること）
**音楽への愛を言葉で記録し、共有する**
- 自分が感じた音楽の魅力を言語化できる
- 深掘りした内容を書ける

### 🚀 Skill（どう届けるか）
**学びながら、技術で届ける**

1. **SEO** - 検索で見つけてもらえる構造を作る
   - キーワード戦略、メタデータ最適化
   - データを見ながら改善

2. **Next.js** - サイト運営を通して学ぶ（学習中）
   - 実装しながら理解を深める
   - 毎日5分の小課題で継続

3. **デザイン** - 読みやすさと美しさを追求（学習中）
   - 余白、配色、タイポグラフィ
   - 実践を通して感覚を磨く

## 🚀 毎日の作業

### 30分チャレンジを開始

```
今日の作業を開始します
```

このコマンドで以下が自動実行されます：
1. **データ分析（5分）** - Search Console/Analytics確認、Issue提案
2. **記事改善（10分）** - 既存記事1本を改善
3. **新規記事（10分）** - 新しい記事を1本作成
4. **Next.js課題（5分）** - 学習項目を1つ実装

**詳細**: [毎日の作業ルーチン](./docs/DAILY-ROUTINE.md)
